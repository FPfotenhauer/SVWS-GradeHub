# ADR-0003: Routing-Strategie — Vue Router Hash Mode vs. History Mode

| Feld             | Wert                                              |
|------------------|---------------------------------------------------|
| **ID**           | ADR-0003                                          |
| **Titel**        | Routing-Strategie — Vue Router Hash Mode vs. History Mode |
| **Status**       | Akzeptiert                                        |
| **Datum**        | 2026-04-04                                        |
| **Autor**        | FPfotenhauer                                      |
| **Projekt**      | SVWS-GradeHub                                     |
| **Abhängig von** | ADR-0001 (Browser-First, Offline-Build), ADR-0005 (Secure Context) |

---

## Kontext

GradeHub ist eine Single-Page-Application (SPA) mit Vue Router. Für SPAs gibt
es zwei grundlegend verschiedene Routing-Strategien:

**History Mode** nutzt die HTML5 History API und erzeugt "saubere" URLs:
```
https://gradehub.schule.de/lerngruppen/42/noten
```
Der Browser sendet beim direkten Aufruf dieser URL eine echte HTTP-Anfrage
an den Server. Der Server muss deshalb **alle Routen auf `index.html`
umleiten** — sonst antwortet er mit `404 Not Found`.

**Hash Mode** kodiert die Route im URL-Fragment nach `#`:
```
https://gradehub.schule.de/#/lerngruppen/42/noten
file:///C:/GradeHub/index.html#/lerngruppen/42/noten
```
Der Browser sendet den Teil nach `#` **nicht** an den Server. Die Navigation
findet ausschließlich im Browser statt — kein Server-Eingriff nötig.

### Der entscheidende Constraint aus ADR-0001

GradeHub muss in zwei Deployment-Varianten funktionieren:

| Variante | URL-Schema | Server |
|----------|------------|--------|
| Webserver (nginx, Apache, GitHub Pages) | `https://` | vorhanden |
| Direktaufruf Offline-Build | `file:///` | **keiner** |

Im `file://`-Modus gibt es schlicht keinen HTTP-Server, der eine
Weiterleitungsregel ausführen könnte. History Mode scheidet damit
als alleinige Strategie aus.

### Weiterer Constraint aus ADR-0005

Die Browser-Verschlüsselung (`window.crypto.subtle`) erfordert einen
**Secure Context** — also `https://` oder `localhost`. Im `file://`-Modus
ist `crypto.subtle` in den meisten Browsern nicht verfügbar. Das bedeutet:
die Verschlüsselungsfunktion ist ohnehin auf den Webserver-Betrieb beschränkt,
unabhängig vom Routing.

---

## Entscheidung

### Hash Mode als einheitliche Routing-Strategie

GradeHub verwendet **ausschließlich den Hash Mode** von Vue Router:

```typescript
// src/router/index.ts
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    { path: '/',                    component: StartView },
    { path: '/verbinden',           component: VerbindungsView },
    { path: '/lerngruppen',         component: LerngruppenView },
    { path: '/lerngruppen/:id',     component: NotenEingabeView },
    { path: '/export',              component: ExportView },
  ]
})
```

**Ein Router, eine Konfiguration, beide Deployment-Varianten funktionieren
ohne Anpassung.**

---

## Begründung

### Warum nicht History Mode?

| Anforderung | History Mode | Hash Mode |
|-------------|-------------|-----------|
| `file://`-Offline-Betrieb | ✗ — Server-Weiterleitung nötig | ✓ — kein Server nötig |
| Webserver-Betrieb | ✓ — mit Konfiguration | ✓ — ohne Konfiguration |
| GitHub Pages Deployment | ✗ — erfordert `404.html`-Hack | ✓ — funktioniert nativ |
| Direkter URL-Aufruf (Bookmark, Link) | ✗ — ohne Server-Config `404` | ✓ — immer funktionsfähig |
| Serverseitige Konfiguration nötig | Ja (nginx/Apache Rewrite-Rule) | Nein |

Der zentrale Punkt: GradeHub soll auch ohne jegliche Server-Infrastruktur
funktionieren. History Mode würde Schulen zwingen, ihren Webserver korrekt
zu konfigurieren — eine Einstiegshürde, die ADR-0001 explizit vermeiden will.

### Warum ist Hash Mode für GradeHub akzeptabel?

SPAs mit öffentlichem Publikum (Marketing-Seiten, SEO-relevante Inhalte)
profitieren von History Mode, weil Suchmaschinen-Crawler `#`-Fragmente
traditionell ignorieren. Für GradeHub gelten diese Argumente nicht:

- **Keine SEO-Anforderung** — GradeHub ist eine Fachanwendung für
  authentifizierte Lehrkräfte, kein öffentliches Webangebot
- **Keine serverseitige Rendering-Anforderung** — alle Daten kommen
  per REST oder Datei-Upload, kein Server-Side-Rendering
- **Deep-Links sind intern** — URLs werden nicht extern geteilt;
  die Lehrkraft navigiert innerhalb der Anwendung

### Verhalten beim Seitenaufruf / Bookmark

Im Hash Mode funktionieren Bookmarks und direkte URL-Aufrufe zuverlässig
in beiden Deployment-Varianten:

```
# Webserver
https://gradehub.schule.de/#/lerngruppen/42   → lädt index.html, Router
                                                 navigiert zu /lerngruppen/42

# Offline
file:///C:/GradeHub/dist/index.html#/lerngruppen/42  → identisch
```

---

## Konsequenzen für den Webserver-Betrieb

Im Webserver-Deployment wird **keine** Rewrite-Regel benötigt. Eine minimale
nginx-Konfiguration genügt:

```nginx
server {
    listen 443 ssl;
    server_name gradehub.schule.de;

    root /var/www/gradehub/dist;
    index index.html;

    # Alle Requests → index.html (für Hash Mode eigentlich nicht nötig,
    # aber schadet nicht und macht den Server robust gegen Tippfehler)
    location / {
        try_files $uri $uri/ /index.html;
    }

    # CORS für SVWS-Server-Verbindung (falls Reverse Proxy genutzt)
    location /api/ {
        proxy_pass http://svws-server:8443/;
        add_header 'Access-Control-Allow-Origin' 'https://gradehub.schule.de' always;
        add_header 'Access-Control-Allow-Headers' 'Authorization, Content-Type' always;
        add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS' always;
    }
}
```

Für **GitHub Pages** (wie bei SVWS-Conference) funktioniert Hash Mode
nativ ohne jegliche Konfiguration.

---

## Konsequenzen für den Offline-Build

Der Offline-Build (`npm run build`) aus SVWS-Conference wird analog
übernommen. Hash Mode stellt sicher, dass `dist/index.html` per
Doppelklick direkt geöffnet werden kann:

```
dist/
  index.html          ← Doppelklick öffnet die App
  assets/
    app.[hash].js
    app.[hash].css
```

**Hinweis im UI:** GradeHub erkennt den `file://`-Modus beim Start
(`window.location.protocol === 'file:'`) und blendet einen Hinweis ein,
dass die Verschlüsselungsfunktion (ADR-0005) in diesem Modus nicht
verfügbar ist.

```typescript
// src/composables/useDeploymentMode.ts
export function useDeploymentMode() {
  const isFileProtocol = window.location.protocol === 'file:'
  const isSecureContext = window.isSecureContext

  return {
    isFileProtocol,
    isSecureContext,
    encryptionAvailable: isSecureContext && !isFileProtocol,
    onlineAvailable: !isFileProtocol,
  }
}
```

## Verworfene Alternativen

| Alternative | Grund für Ablehnung |
|-------------|---------------------|
| **History Mode einheitlich** | Bricht den `file://`-Offline-Betrieb; erfordert Server-Konfiguration |
| **History Mode im Webserver, Hash Mode im Offline-Build (zwei Builds)** | Doppelter Build-Aufwand; zwei Router-Konfigurationen zu pflegen; unterschiedliches Verhalten je nach Deployment schwer dokumentierbar |
| **Memory Mode** (`createMemoryHistory`) | Kein Bookmark-Support, keine Browser-Navigation (Vor/Zurück); ausschließlich für SSR/Testing geeignet |
| **Kein Router (manuelle Navigation)** | Widerspricht dem Vue-3-Architekturmuster; kein Deep-Linking möglich |

---

## Zusammenfassung

```
Anforderung: file://-Offline-Betrieb  ──┐
Anforderung: Webserver-Betrieb        ──┼──► Hash Mode  ✓
Anforderung: GitHub Pages             ──┘
Keine SEO-Anforderung                 ──► Hash-URLs akzeptabel ✓
Keine Server-Konfiguration            ──► Niedrige Einstiegshürde ✓
```
